# **TECHNICAL ASSIGNMENT**
## **Prototype Design – Loyalty System Dashboard**
**Client:** “Ideal Body Studio”
**Project Type:** Internal Client Loyalty Web App
**Priority:** Mobile-first design
## **1. Project Overview**
The goal is to design a web-based client dashboard for the internal loyalty system of “Ideal Body Studio”. 
The dashboard must clearly visualize:
- Client loyalty status
- Financial benefits received
- Visit statistics
- Personal pricing
- Progress toward the next status level
The interface should feel premium, structured, motivating, and emotionally reinforcing.
Primary focus: **mobile version**
Secondary: **desktop adaptation**
## **2. Design Style & Positioning**
Target audience: women 30–55 years old.
The interface should feel:
- Premium but not aggressive
- Clean and minimal
- Trustworthy
- Motivating
- Elegant and calm
Avoid:
- Overly bright colors
- “Cheap” loyalty-style gamification visuals
- Too much clutter
The emotional goal
The client should feel valued, rewarded, and “special”.
## **3. Layout Structure**
### **3.1 General Layout**
- Left navigation sidebar (collapsible on desktop, bottom navigation on mobile) 
- Top navbar
- Main dashboard content area
- Footer
Logo and brand colors will be provided separately.
---
## **4. Dashboard Structure (Main Screen)**
The dashboard must follow a **data-driven layout (dashboard principle)**.
### **4.1 Status Block (Top Priority Section)**
Display prominently:
- **Your Status: PLATINUM**
- Individual Discount: 12%
- To next level: 8,500 ₽
- Progress bar visualization
This block must feel visually dominant.
The progress bar should clearly show advancement toward the next level.
### **4.2 Financial Benefits Block**
Display as separate card section:
- You paid in the studio: 128,000 ₽
- Discounts received: 21,340 ₽
- Saved: 16.6%
Important:
Savings percentage should be visually emphasized.
The idea: the client clearly sees how much she benefits financially.
---
### **4.3 Activity Statistics Block**
Bullet-style or card-style metrics:
- Visits: 37
- Completed courses: 4
- Average interval: 6 days
Should look structured and easy to scan.
### **4.4 Personal Pricing Block**
Separate card with highlight:
Your Individual Pricing:
- LPG – 1,450 ₽ (instead of 1,700 ₽
- Manual massage – 1,000 ₽ (fixed)
The “instead of” price should be visually differentiated (crossed out or lighter color)
This block must reinforce exclusivity.
---
## **5. Left Navigation Panel**
Desktop: vertical sidebar
Mobile: bottom navigation or hamburger menu
Include the following sections:
1. Dashboard (default page)
2. Book Appointment
3. Procedure Calendar
4. Visit History Calendar
5. AI Assistant
6. Nutrition Coach
7. Physical Activity Coach
Icons must be elegant and minimal.

---

Designer must create basic UI prototypes for:
### **6.1 Book Appointment Page**
- Service selection
- Date & time selection
- Confirmation state
### **6.2 Procedure Calendar**
- Upcoming appointments
- Status indicators (completed / upcoming / cancelled)

### **6.3 Visit History Calendar**
- Monthly calendar view
- Marked visit days

### **6.4 AI Communication Section**
Chat-style interface.
Tabs inside:
- AI Assistant
- Nutrition Coach
- Physical Activity Coach
Chat interface should feel clean and premium (not like a generic chatbot).
---
## **7. Mobile Version (High Priority)**
Mobile-first approach:
- Bottom navigation preferred
- Large readable typography
- Clear spacing
- Cards stacked vertically
- No information overload
All dashboard blocks must be easily scrollable.
---
## **8. Desktop Version**
- Sidebar on the left
- Dashboard grid layout
- Balanced whitespace
- More horizontal data distribution
---
## **9. Footer**
Include:
- Studio contact information

Minimal design.

## **10. Visual Requirements**
Designer must provide:
- Full mobile prototype (primary)
- Desktop adaptation
- UI kit components (buttons, cards, inputs, progress bars)
- Status badge designs (Bronze, Silver, Gold, Platinum)

## **11. UX Goals**
The system must:
- Visually show financial benefit
- Reinforce loyalty status
- Motivate progression
- Feel exclusive
- Be intuitive without instructions
The client should understand in under 5 seconds:
“I am getting real value here.”
